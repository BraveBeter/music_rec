### 完成事项

#### 1. Admin 前端重设计（b62afa9）

**布局重构**：将原来铺满浏览器的单页 Dashboard 拆分为侧边栏 + 内容区的经典管理面板布局。

- 固定左侧边栏（220px），深色背景 #0f0f23，包含导航菜单和用户信息
- 内容区 max-width 1200px 居中，不铺满浏览器
- 将 Vue Router 改为嵌套路由，AppLayout 作为父路由组件

**页面拆分**（从单一 Dashboard.vue 拆为 5 个独立视图）：

| 页面 | 功能 |
|------|------|
| Dashboard | 概览：统计卡片 + 模型状态 + 最近训练 |
| DataImport | 数据导入：Jamendo/Deezer/LastFM/合成数据 |
| Training | 模型训练：实时进度可视化 + 日志流 + 历史记录 |
| Scheduler | 定时任务：调度管理 + 阈值配置 |
| Models | 模型状态：可用性 + 训练指标 |

**新增共享组件**：StatCard、ProgressBar、LogPanel、StatusBadge

**新增 Pinia Store**：training.ts（SSE 实时进度）、scheduler.ts（调度管理）

#### 2. 训练进度可视化（后端 + 前端）

**后端**：
- 新建 `ml_pipeline/training/progress.py`：基于 JSON 文件的跨进程进度追踪（ProgressTracker），支持原子写入
- 修改 4 个训练脚本（train_sasrec、train_deepfm、train_baseline、preprocess），支持 `--task-id` 参数，实时写入进度
- 新建 `admin/services/training_service.py`：训练编排服务
- 重写 `admin/api/training.py`：新增 SSE 流式推送、进度查询、历史记录、取消训练等端点

**前端**：
- Training.vue 通过 SSE 连接实时显示 epoch 进度条、loss 值、训练日志流
- 训练历史表格展示已完成/出错/中断的训练记录

#### 3. 定时任务调度系统

**数据库**：
- 新增 `training_schedules` 表：存储调度任务配置（cron/interval/threshold 三种模式）
- 新增 `training_threshold_state` 表：追踪阈值触发状态
- 启动时自动建表（`Base.metadata.create_all`），兼容已存在的数据库

**后端**：
- 新建 `admin/services/scheduler_service.py`：APScheduler 封装，支持：
  - Cron 表达式调度
  - 固定间隔调度
  - 数据量阈值触发（每 10 分钟检查交互数增量）
- 新建 `admin/api/scheduler.py`：调度 CRUD API + 阈值配置 API
- 全局训练锁（asyncio.Lock）：确保同一时间只有一个训练流程运行
- `max_instances=1` + `coalesce=True`：防止同一任务重复触发

**前端**：
- Scheduler.vue：调度任务列表、新建/删除/启禁用、阈值配置表单
